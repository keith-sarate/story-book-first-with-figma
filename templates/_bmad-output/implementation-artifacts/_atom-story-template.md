## Story

As a **demo presenter**,
I want **the `{{AtomName}}` atom implemented as a React component with a Storybook story, consuming design tokens only**,
so that **I can show this atom live in Storybook before composing it into molecules**.

## Acceptance Criteria

1. `src/components/atoms/{{AtomName}}/{{AtomName}}.tsx` exists, with typed props mirroring the Figma variant set on node `{{atom_node_id}}`.
2. `src/components/atoms/{{AtomName}}/{{AtomName}}.stories.tsx` exists with `Default` plus one story per Figma-documented variant/state.
3. `src/components/atoms/{{AtomName}}/index.ts` re-exports the component (named + default).
4. No hardcoded color/spacing/radius/typography values — only Tailwind utilities backed by `src/tokens/tokens.ts`.
5. The atom does **not** import any other atom (atoms are leaves).
6. Storybook a11y addon shows zero critical violations on every story of this atom.
7. The atom renders in `npm run storybook` without console errors or warnings.
8. **Visual Validation Loop passed for every story of this atom**: `pnpm visual:check` produced `actual.png` and `expected.png` per story, the dev walked the full Visual Checklist from [docs/workflows/storybook-first-with-figma.md](../../docs/workflows/storybook-first-with-figma.md#visual-validation-loop), and every item is ticked. Drift fixed and re-validated until clean. Outcome (what matched, what was fixed) recorded in `Completion Notes List` below.
9. **No Figma sync in this story** — bulk sync happens in `1-2-99-atoms-sync.md` once all per-atom stories are done.

## Dev Notes

- **Demo cadence:** this story is intentionally small (~5–10 min). The point is to show one atom go from Figma → Storybook live during the presentation.
- **Stay in-scope:** if Figma shows the atom *used inside* a layout, do NOT replicate the layout. Build the atom only; layout context is a molecule's job.
- **Token discipline:** if you cannot express a value with a token, ask the user before introducing a one-off — don't silently extend the token set mid-atom.
